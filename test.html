CSS Safe Area Insets

Safe area insets ensure your content is displayed correctly on modern devices with notches, rounded corners, or interactive elements that might overlap your content.

Demo Element with Safe Area Bottom Padding

The green box below uses safe area insets to prevent content from being hidden behind browser UI or device notches.

This element has safe area bottom padding applied.
On devices with bottom UI (like iPhone home indicator), extra padding will be added automatically.
/* How to use safe area insets */
.element {
  padding-bottom: env(safe-area-inset-bottom, 30px); /* Modern browsers */
  padding-bottom: constant(safe-area-inset-bottom, 30px); /* Older iOS support */
}

/* The second value (30px) is a fallback for browsers that don't support safe area insets */
Without Safe Area Insets

Content might be hidden behind:

  • Mobile browser UI
  • Device notches
  • Home indicators
  • Navigation bars
With Safe Area Insets

Content is properly padded to avoid:

  • UI overlapping
  • Hard-to-reach elements
  • Content being hidden
  • Poor user experience

Other Safe Area Properties

You can use safe area insets for all four sides of your content:

.full-safe-area {
  padding-top: env(safe-area-inset-top);
  padding-right: env(safe-area-inset-right);
  padding-bottom: env(safe-area-inset-bottom);
  padding-left: env(safe-area-inset-left);
}
